Este asyncScheduler NO crea un Observable si no una Subscription

  • Subscription: recordemos que un Subscription no es mas que el producto de cuando ejecutamos un subscribe() de un Observable


asyncScheduler: Nos permite trabajar de forma similar a cuando trabajamos con: 

  
    {{ 'setTimeout(() => {}, 3000);' }}
    {{ 'setInterval(() => {}, 3000);' }}


Para empezar a trabajar con asyncScheduler necesitamos mandar 2 argumentos:

  • 1 Funcion: El primer argumento es la funcion que ejecutara nuestro schedule
  • 2 Tiempo/Delay: El tiempo/delay que queremos que tome para que se empiece a ejecutar
  • 3 Estado: en esta parte solo podemos mandar 1 argumento, por lo cual si necesitamos de mas info, tenemos que pasar un Objeto 
          
        {{ stringCode }}
